# Техническое задание: AI Team Office

## 1. Общее описание

### Назначение системы
AI Team Office — интерактивная визуализация команды AI-агентов в формате виртуального офиса. Система отображает текущий статус, задачи и активность агентов в реальном времени, создавая наглядное представление о работе команды.

### Целевая аудитория
- Руководители проектов — для мониторинга загрузки команды
- Разработчики — для отслеживания статуса коллег
- Стейкхолдеры — для визуализации прогресса проектов

### Основные функции
- Визуализация агентов в рабочей зоне и комнате отдыха
- Отображение текущих задач и прогресса
- Интеграция с GitHub Issues
- Real-time обновление статусов
- Детальная информация об агенте по клику

## 2. Требования к функциональности

### 2.1 Отображение агентов в рабочей зоне
- 6 рабочих столов с 3D-эффектом
- Каждый стол оборудован монитором
- На мониторе отображается текущая задача агента
- Агенты со статусом `working` размещаются за столами

### 2.2 Отображение агентов в комнате отдыха
- Отдельная зона для агентов со статусом `resting`
- Визуально отличается от рабочей зоны
- Агенты автоматически перемещаются при смене статуса

### 2.3 Статус агентов
- Два состояния: `working` (работает) / `resting` (отдыхает)
- Визуальная индикация статуса (цвет, анимация)
- Автоматическое обновление при изменении

### 2.4 Текущие задачи агентов
- Отображение `currentTask` на мониторе агента
- Прогресс-бар с процентом выполнения (`progress`)
- Название проекта (`project`)

### 2.5 GitHub Issues агентов
- Список issues, назначенных агенту
- Отображение в модальном окне
- Статус issues (open/closed)

### 2.6 Обновление в реальном времени
- WebSocket соединение для мгновенных обновлений
- Без перезагрузки страницы
- Плавные анимации при перемещении агентов

## 3. Технические требования

### 3.1 Frontend
- **Стек:** HTML5, CSS3, vanilla JavaScript
- **CSS:** 3D-трансформации для столов и мониторов
- **WebSocket:** нативный WebSocket API для real-time коммуникации
- **Адаптивность:** корректное отображение на разных разрешениях

### 3.2 Backend
- **Платформа:** Node.js
- **WebSocket сервер:** Socket.io
- **HTTP API:** Express.js
- **CORS:** настроен для доступа с фронтенда

### 3.3 API Endpoints

#### GET `/api/agents/status`
Возвращает массив всех агентов с текущим статусом.

**Response:**
```json
{
  "agents": [
    {
      "id": "agent-001",
      "name": "UI Designer",
      "role": "UI/UX Designer",
      "status": "working",
      "emoji": "🎨",
      "currentTask": "Design login page",
      "project": "Meetify",
      "progress": 75,
      "deskId": 1,
      "issues": ["#42", "#43"]
    }
  ]
}
```

### 3.4 WebSocket Events

#### Client → Server
- `subscribe` — подписка на обновления

#### Server → Client
- `agent:update` — обновление данных агента
- `agent:statusChanged` — изменение статуса агента
- `agent:taskUpdated` — обновление задачи

## 4. Структура данных агента

```javascript
{
  // Уникальный идентификатор агента
  id: string,
  
  // Отображаемое имя
  name: string,
  
  // Роль в команде
  role: string,
  
  // Текущий статус: 'working' | 'resting'
  status: 'working' | 'resting',
  
  // Эмодзи-аватар агента
  emoji: string,
  
  // Текущая выполняемая задача
  currentTask: string,
  
  // Название проекта
  project: string,
  
  // Прогресс выполнения (0-100)
  progress: number,
  
  // Номер рабочего стола (1-6)
  deskId: number,
  
  // Список GitHub issues
  issues: Array<string>
}
```

## 5. Интерфейс

### 5.1 Рабочая зона
- **Расположение:** центральная часть экрана
- **Конфигурация:** 6 столов (2 ряда по 3)
- **Визуальный стиль:** 3D-столы с перспективой, мониторы с подсветкой
- **Каждый стол содержит:**
  - Монитор с отображением задачи
  - Имя агента
  - Эмодзи-аватар
  - Индикатор прогресса

### 5.2 Комната отдыха
- **Расположение:** боковая/нижняя часть экрана
- **Визуальный стиль:** отличная от рабочей зоны цветовая гамма (теплые тона)
- **Элементы:** диваны, кресла, растения
- **Агенты:** отображаются в расслабленной позе

### 5.3 Модальное окно агента
Открывается при клике на агента:
- Полное имя и роль
- Эмодзи-аватар (крупный)
- Текущий статус с индикатором
- Полное описание текущей задачи
- Прогресс-бар с процентом
- Список GitHub issues со ссылками
- История недавней активности (опционально)

## 6. Интеграции

### 6.1 GitHub API
- **Назначение:** получение issues, назначенных агентам
- **Аутентификация:** GitHub Personal Access Token
- **Endpoints:**
  - `GET /repos/{owner}/{repo}/issues?assignee={agent}`
- **Обновление:** polling каждые 5 минут или по webhook

### 6.2 WebSocket
- **Протокол:** Socket.io
- **Транспорт:** WebSocket с fallback на long-polling
- **Функционал:**
  - Broadcast обновлений всем подключенным клиентам
  - Комнаты для фильтрации по проектам (опционально)
  - Reconnect с восстановлением состояния
